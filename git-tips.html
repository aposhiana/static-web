My Amazing Git Cheat Sheet

Setting Up a Repo

  1. cd to the directory where you want the git repository to be
  2. git status (do this to ensure you don't already have a git repository)
  3. git init
  4. Create a new repository in Bitbucket or GitHub
  5. git remote add origin [your-remote-repo-url]
  6. git add .
  7. git commit -m "first commit"
  8. git push -u origin master

Commands to Remember

Some of the following descriptions were taken from the GitHub Git Cheat Sheet .

fetch [remote-shortname]
Fetches all new data from the specified remote without merging
push [remote-shortname] [branch]
Attempts to push to the specified remote branch. Note that adding the -u argument will also set up argumentless git pull for that branch
merge [branch]
Attempts to merge specified branch with the current branch. Note that to merge a remote branch that has been fetched we run git fetch remotename/branchname.
reset [commit]
Changes HEAD to point to specified commit (the branch that HEAD was already pointing to stays the same). Next, it updates the index to match the commit where the new HEAD is pointing to. It does not change the contents of the working directory.
reset --hard [commit]
Does the same as git reset but also changes the contents of the working directory to match the specified commit. This will cause any uncommitted changes to be lost.
branch
Lists all branches
checkout -b [new-branch]
Creates a new branch and checks it out
checkout [branch]
Switches HEAD to point to the specified branch
branch -d [branch]
Deletes the specified branch
stash
Temporarily stores all modified tracked files to a stack
stash pop
Applies the stash and deletes it from the stack
stash list
Lists all stashes stored
rm --cached [file]
Removes the specified file from version control without deleting the local file
status
Lists all new or modified files to be committed
diff
Lists file differences not yet staged
remote -v
Lists the names and URLs of all your remote handles
remote add [remote-shortname] [your-remote-repo-url]
Adds a new remote with the specified info
remote remove [remote-shortname]
Removes all information about a remote
