Useful Commands in Unix

grep

Grep stands for globally search for regex and print. Use the command to simply search for a string as grep stringtofind and then include any other arguments. It can be run simply to search in the current directory or it can be run as grep stringtofind somedir/filename to search a specific file for a string. Additional arguments are listed in the table below with brief descriptions about their behaviors.

-r
Recursively - use this when searching a directory for example: . (the current directory)
-i
Ignore case of letters
-w
Ignore substrings
-l
Only list the file names where the matches occur
-I
Ignore binary files
-n
Print line numbers
--include=*.html
Search only files with the .html extension (could be another extension)

cat

Cat is short for concatenate. It can be used in this manner: cat file1.txt file2.txt file3.txt > out-file.txt. Note that > overwrites out-file.html whereas >> does not.

cat * > out-file.txt
Concatenates all files in the current directory
cat *.txt >> out-file.txt
Concatenates all files in the current directory with the specified file extension (in this case .txt)

rmdir

rmdir can be used to remove an empty directory
